# Contributing to the Academic Website Template

Thank you for your interest in improving this academic website template! This guide will help you contribute.

## How to Contribute

### Reporting Issues

If you find a bug or have a suggestion:

1. Check if the issue already exists in the [Issues](../../issues) section
2. If not, create a new issue with:
   - A clear title
   - Detailed description
   - Steps to reproduce (for bugs)
   - Expected vs actual behavior

### Suggesting Enhancements

We welcome suggestions for new features or improvements:

1. Open an issue describing the enhancement
2. Explain the use case and benefits
3. If possible, provide examples from other academic websites

### Submitting Changes

1. Fork the repository
2. Create a new branch: `git checkout -b feature/your-feature-name`
3. Make your changes
4. Test your changes locally with `bundle exec jekyll serve`
5. Commit with clear messages: `git commit -m "Add feature: description"`
6. Push to your fork: `git push origin feature/your-feature-name`
7. Open a Pull Request

## Development Guidelines

### File Organization

- `_pages/` - Main content pages (About, Research, etc.)
- `_posts/` - Blog posts
- `_data/` - Data files (navigation, publications, etc.)
- `assets/` - Images, files, CSS, JS
- `_config.yml` - Site configuration

### Coding Standards

- Use clear, descriptive variable names
- Follow Jekyll/Liquid template conventions
- Keep YAML files properly indented (2 spaces)
- Add comments for complex configurations

### Testing

Before submitting:

1. Build the site locally: `bundle exec jekyll serve`
2. Check all pages load correctly
3. Verify navigation works
4. Test on mobile and desktop
5. Validate YAML syntax

### Documentation

When adding features:

- Update README.md if needed
- Update SETUP.md with configuration instructions
- Add inline comments for complex code
- Include examples in documentation

## Types of Contributions Welcome

### Content Improvements

- Better example text
- More comprehensive templates
- Additional academic fields/disciplines
- Improved documentation

### Feature Additions

- New page templates (e.g., Talks, Awards)
- Additional data file examples
- New sections or layouts
- Integration with academic tools

### Bug Fixes

- Broken links
- Layout issues
- Configuration errors
- Compatibility problems

### Design Enhancements

- Better responsive design
- Accessibility improvements
- Theme customizations
- Visual refinements

## Questions?

If you have questions about contributing:

- Open an issue with the "question" label
- Check existing issues and pull requests
- Review the Jekyll and Minimal Mistakes documentation

## Code of Conduct

- Be respectful and constructive
- Welcome newcomers
- Focus on what's best for the community
- Show empathy towards others
